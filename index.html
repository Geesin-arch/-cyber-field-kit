The one rule that carries across all three volumes

Accessing any computer, account, network, or website without the owner's written permission is illegal in almost every country, regardless of intent. Every tool and command in this kit is standard, publicly documented syntax — what makes usage legal is never the command, it's written authorization for that specific target.

Publish this kit with GitHub Pages

These four files are a complete, self-contained static site — no build step, no dependencies. This puts it at a free public URL in a few minutes.

Create a repository

On GitHub, click New repository, give it a name (e.g. cyber-field-kit), and keep it public.

Add the four files

Upload index.html, field-manual.html, armory.html, and cheatsheets.html to the repository root (drag-and-drop works on the "Add file → Upload files" screen).

Turn on Pages

Go to Settings → Pages, set Source to Deploy from a branch, pick the main branch and / (root) folder, then Save.

Visit your site

After a minute, GitHub shows your live URL at the top of the Pages settings — typically https://your-username.github.io/cyber-field-kit/.
